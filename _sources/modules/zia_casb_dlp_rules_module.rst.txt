zscaler.ziacloud.zia_casb_dlp_rules module -- Manages CASB DLP rules
++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++

.. Collection note

.. note::
    This module is part of the `zscaler.ziacloud collection <https://galaxy.ansible.com/ui/repo/published/zscaler/ziacloud/>`_ (version 2.2.0).

    It is not included in ``ansible-core``.
    To check whether it is installed, run :code:`ansible-galaxy collection list`.

    To install it, use: :code:`ansible\-galaxy collection install zscaler.ziacloud`.
    You need further requirements to be able to use this module,
    see :ref:`Requirements <ansible_collections.zscaler.ziacloud.zia_casb_dlp_rules_module_requirements>` for details.

    To use it in a playbook, specify: :code:`zscaler.ziacloud.zia_casb_dlp_rules`.

Synopsis

.. Description

- Adds, updates, or removes SaaS Security Data at Rest Scanning DLP rules.

Notes

   - :literal:`type` and :literal:`name` are required for create. :literal:`type` with :literal:`id` or :literal:`name` for update/delete.

.. Examples

Examples
--------

.. code-block:: yaml+jinja

    - name: Create a CASB DLP rule
      zscaler.ziacloud.zia_casb_dlp_rules:
        provider: '{{ provider }}'
        name: "My DLP Rule"
        type: OFLCASB_DLP_ITSM
        order: 1
        description: "Rule created by Ansible"
        action: OFLCASB_DLP_REPORT_INCIDENT
        severity: RULE_SEVERITY_HIGH
        components:
          - COMPONENT_ITSM_OBJECTS
          - COMPONENT_ITSM_ATTACHMENTS
        collaboration_scope:
          - ANY
        file_types:
          - FTCATEGORY_APPX
          - FTCATEGORY_SQL

    - name: Update a CASB DLP rule by ID
      zscaler.ziacloud.zia_casb_dlp_rules:
        provider: '{{ provider }}'
        id: 1070199
        type: OFLCASB_DLP_ITSM
        name: "Updated Rule Name"
        order: 1

    - name: Delete a CASB DLP rule
      zscaler.ziacloud.zia_casb_dlp_rules:
        provider: '{{ provider }}'
        id: 1070199
        type: OFLCASB_DLP_ITSM
        state: absent
